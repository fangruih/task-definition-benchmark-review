展开 Benchmark Task 类型 原文整句 quote 他们怎么定义 task Judge / oracle 设计
Terminal-Bench Terminal agent task
“Each task features a unique environment, human-written solution, and comprehensive tests for verification.”
Terminal-Bench paper
  • 独立 Linux terminal 环境
  • 人类写的任务说明和解法
  • 验证最终状态的测试
  • 强调真实、困难、长程命令行工作
  • 运行测试脚本
  • 检查文件、命令输出、程序行为、服务或环境状态
  • 不固定解题路径
  • 核心是最终状态可验证
CocoaBench Unified digital agent task
“Tasks are specified only by an instruction and an automatic evaluation function over the final output.”
CocoaBench paper
  • 输入是 instruction
  • 输出由 final-output evaluator 检查
  • 任务需要组合视觉、搜索、代码能力
  • 强调长程数字任务
  • 自动 evaluation function
  • 只评估 final output
  • 不绑定工具轨迹
  • 不要求固定 workflow
OSWorld Computer-use task
“OSWorld can serve as a unified, integrated computer environment for assessing open-ended computer tasks that involve arbitrary applications.”
OSWorld paper
  • 真实 OS / app 环境
  • 开放式电脑操作任务
  • 包含初始状态和自然语言目标
  • 包含可交互应用和任务特定配置
  • custom execution-based evaluation script
  • 检查系统、应用、文件或配置状态
  • 评估最终状态
  • 不把点击序列当作唯一答案
SWE-bench Software engineering issue
“We find real-world software engineering to be a rich, sustainable, and challenging testbed for evaluating the next generation of language models.”
SWE-bench paper
  • instance 来自真实 GitHub issue / PR
  • 输入是 codebase 和 issue 描述
  • 输出是修改代码的 patch
  • 目标是解决真实软件工程问题
  • 运行测试集
  • 判断 patch 是否修复 issue
  • 允许多种正确实现
  • 以行为是否通过 regression / hidden tests 为准
GAIA General assistant question
“GAIA proposes real-world questions that require a set of fundamental abilities such as reasoning, multi-modality handling, web browsing, and generally tool-use proficiency.”
GAIA paper
  • 现实问题
  • 需要推理、工具、浏览或多模态处理
  • 最终答案明确、短小、可比对
  • 过程开放，答案封闭
  • 答案匹配
  • 等价性判断
  • 重点是最终答案正确性
  • 不评估搜索路径本身
PaperBench Research replication task
“Agents must replicate 20 ICML 2024 Spotlight and Oral papers from scratch, including understanding paper contributions, developing a codebase, and successfully executing experiments.”
PaperBench paper
  • 复现 AI research paper
  • 从零理解贡献、开发代码、执行实验
  • 复杂 deliverable
  • 拆成子任务和明确评分点
  • 分层 rubric
  • 支持 partial credit
  • 评估代码、实验、结果、文档
  • 不是单个 pass/fail 测试
MLR-Bench Open-ended ML research
“In this work, we introduce MLR-Bench, a comprehensive benchmark for evaluating AI agents on open-ended machine learning research.”
MLR-Bench paper
  • 开放式 ML research
  • 覆盖 idea generation、proposal、experimentation、paper writing
  • 目标是研究产物
  • 不是单一答案任务
  • stagewise assessment
  • final paper evaluation
  • 关注 idea、proposal、experiment、writing
  • 特别检查实验有效性和结果可信度
Agents' Last Exam Economic long-horizon workflow
“This paper introduces Agents' Last Exam (ALE), a benchmark designed to evaluate AI agents on long-horizon, economically valuable, real-world tasks with verifiable outcomes.”
Agents' Last Exam paper
  • 长程专业工作流
  • 经济价值高、真实世界任务
  • 覆盖非体力行业
  • 按 O*NET / SOC 2018 组织任务分类
  • 强调 verifiable outcomes
  • 报告 full pass rate
  • 评估最终专业工作结果
  • 不是单步问答评测
ClawBench Live web task
“Everyday online tasks offer a realistic yet unsolved testbed for evaluating the next generation of AI agents.”
ClawBench paper
  • 日常生活和工作中的线上任务
  • 运行在真实生产网站
  • 包括购买、预约、职位申请、表单填写
  • 覆盖多平台多类别
  • 两阶段 scoring：Stage 1 deterministic interception，Stage 2 LLM judge
  • Stage 1：sandbox request interceptor 按 task.json 的 eval_schema.url_pattern + method 匹配最终 outgoing HTTP request
  • Stage 2：LLM judge 只看 instruction + intercepted request body，返回 match true / false / null
  • final_pass = intercepted AND judge_match is true；不是 unit-test oracle
GDPval / GDPBench Knowledge-work deliverable
“Each GDPval task consists of two primary components: a request (often with reference files) and a deliverable (work product).”
GDPval paper
  • 任务由 request 和 deliverable 组成
  • 经常包含 reference files
  • 基于行业专家真实工作产品
  • 覆盖 44 个职业和高 GDP 行业
  • 论文 headline metric：human expert head-to-head / blinded pairwise comparison
  • 专家看到 request + reference files + 多个匿名 deliverables，并排序/写 justification
  • reproducibility path：开源 220-task gold subset 的 prompts 和 reference files
  • 公开 evals.openai.com automated grading service；论文称 experimental，不是完全等价于专家 oracle
Auto-ClawEval / ClawEnvKit Generated claw-like agent environment
“Using ClawEnvKit, we construct Auto-ClawEval, the first large-scale benchmark for claw-like agents, comprising 1,040 environments across 24 categories.”
ClawEnvKit paper
  • 从自然语言 capability description 生成环境
  • 包含 task specification
  • 包含 tool interface 和 scoring configuration
  • Auto-ClawEval 有 1,040 个环境、24 类
  • validator 检查 feasibility
  • 检查 diversity、structural validity、internal consistency
  • scoring configuration 定义评分
  • completion 是主要评估轴
DeepResearch Bench Deep research report task
“DeepResearch Bench, a benchmark consisting of 100 PhD-level research tasks, each meticulously crafted by domain experts across 22 distinct fields.”
DeepResearch Bench paper
  • 100 个 PhD-level research tasks
  • 由领域专家跨 22 个领域设计
  • agent 需要多步 web exploration、retrieval、synthesis
  • 输出 citation-rich research report
  • RACE：report quality（LLM judge；human alignment evaluated on human-annotated subset）
  • RACE criteria：comprehensiveness / insight-depth / instruction-following / readability（LLM-generated criteria + reference-based scoring）
  • FACT：citation support / effective citations（web scraping + LLM judgment；human alignment reported via README）
  • human validation：human inter-annotator agreement baseline = 68.78%
MMDeepResearch-Bench Multimodal deep research report
“MMDR-Bench emphasizes report-style synthesis with explicit evidence use, where models must connect visual artifacts to sourced claims.”
MMDeepResearch-Bench paper
  • 140 个 expert-crafted tasks
  • 覆盖 21 个领域
  • 每个任务提供 image-text bundle
  • 要求多模态证据驱动的 report-style synthesis
  • FLAE：report quality（LLM judge；human validation not reported in abstract）
  • TRACE：citation-grounded evidence alignment（自动/模型评估；human validation not reported in abstract）
  • MOSAIC：text-visual integrity / visual evidence support（自动/模型评估；human validation not reported in abstract）
DRBench Enterprise deep research task
“DRBench evaluates agents on multi-step queries that require identifying supporting facts from both the public web and private company knowledge base.”
DRBench paper
  • 复杂、开放式 enterprise deep research tasks
  • 任务 grounded in user personas 和 enterprise context
  • 搜索空间包括 public web、company KB、emails、chat、cloud files
  • 输出 coherent, well-structured reports
  • 评估维度：relevant insight recall、factual accuracy、report coherence/structure
  • human-in-the-loop verification：用于 task generation / task quality verification
  • LLM judge：abstract 中未明确说明是否用 LLM-as-judge
  • human validation/agreement：abstract 中未看到 judge-human agreement 数字
ReportBench Academic survey report task
“ReportBench leverages high-quality published survey papers available on arXiv as gold-standard references.”
ReportBench paper
  • 用 survey paper 反推 domain-specific prompts
  • 评估 deep research agent 生成 research report
  • 任务关注学术综述式内容质量
  • 输出需要引用文献并覆盖研究主题
  • Cited literature quality：document type classification（LLM judge；human validation not reported）
  • Cited statements：supporting passage matching + consistency check（LLM judge；human validation not reported）
  • Non-cited claims：web-connected LLM fact-checking + majority vote（LLM judge；human validation not reported）
IDRBench Interactive deep research task
“IDRBench combines a modular multi-agent research framework with on-demand interaction, a scalable reference-grounded user simulator, and an interaction-aware evaluation suite.”
IDRBench paper
  • interactive deep research benchmark
  • 研究目标可能 underspecified 且会演化
  • agent 可以 on-demand interaction
  • 任务包含用户反馈和长文报告生成
  • evaluation suite：interaction-aware，联合评估 quality、alignment、turn/token cost
  • user simulator：reference-grounded scalable user simulator
  • LLM judge：abstract 中未明确说明具体 judge model / LLM-as-judge 机制
  • human validation/agreement：abstract 中未看到 human agreement validation
BrowseComp Deep browsing short-answer task
“BrowseComp comprises 1,266 questions that require persistently navigating the internet in search of hard-to-find, entangled information.”
BrowseComp paper
  • 1,266 个 web browsing questions
  • 需要 persistent internet navigation
  • 查找 hard-to-find, entangled information
  • 最终答案短且可验证
  • predicted answers are short
  • against reference answers 验证
  • 自动或半自动 answer matching
  • 测 deep search persistence 和 creativity
BrowseComp-V3 Multimodal deep browsing task
“The benchmark emphasizes deep, multi-level, and cross-modal multi-hop reasoning.”
BrowseComp-V3 paper
  • 300 个 curated challenging questions
  • 跨 textual 和 visual modalities 搜索证据
  • 强调 deep, multi-level, cross-modal multi-hop reasoning
  • 证据必须 publicly searchable
  • final-answer accuracy
  • expert-validated subgoal-driven process evaluation
  • 分析 intermediate reasoning behaviors
  • 保证可复现的 evidence accessibility
